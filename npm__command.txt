1. Сначала делаем инициализацию через команду npm init и отвечаем на вопросы:

package name - [defender-game"]
version - 1.0.0
description - [Browser based game]
entry point - index.js
test command - оставьте пустым
git repository - [URL вашего GitHub репозитория]
keywords - [game]
author - [ваше имя или псевдоним]
license - ISC

2. Команды для Bable:

npm install --save-dev @babel/core @babel/cli @babel/preset-env
npm install core-js

в package.json в scripts написать "build": "babel src -d dist"

у bable конфигурационный файл создаем сами

НЕ ЗАБЫВАЕМ ПРИ ИСПОЛЬЗОВАНИИ Babel СОЗДАВАТЬ ФАЙЛ .browserslistrc

2.1. При использовании Babel вместе с Webpack нужно скачать пакеты Babel для Webpack:

npm install --save-dev babel-loader

и в конфигурационный файл webpack добавить правило для распознавания Babel


3. Команды для ESlint версии 10>:

npm init @eslint/config@latest

в package.json в scripts написать  "lint": "eslint ."

у ESlint конфигурационный файл создается сам. Мы только добавляем в него игнорирование проверки папки dist и node__module

3. Команды для Webpack:

npm install --save-dev webpack webpack-cli

у Webpack конфигурационный файл создаем сами. В него дальше пишем все что подключаем. Базово подключаем распознавание и создание файлов с расширением .css и .html
для этого прописываем команды пакетов для loader и plugins:

npm install --save-dev html-webpack-plugin
npm install --save-dev mini-css-extract-plugin css-loader

3.1. Для того чтобы когда мы используем Webpack для сборки в папке dist, собранные файлы изменялись в live-режиме, подключим пакет:

npm i -D webpack-dev-server

КОГДА МЫ ИСПОЛЬЗУЕМ СБОРЩИК, ТО НЕ ПИШЕМ В ФАЙЛЕ index.html ПУТИ К css ФАЙЛАМ И script ОН САМ ИХ СОБИРАЕТ

в package.json в scripts написать  "build": "webpack --mode=production", - для билда файлов в папку dist
ГЛАВНЫЙ ФАЙЛ СБОРЩИК В ПАПКЕ src/index.js

в package.json в scripts написать "start": "webpack serve --mode=development" - для запуска live-server-webpack для папки dist
!!!!!! Все что мы добавляем к расширению базового функционала Webpack нужно добавлять в его файл конфигуратор


4. Команды для live-server (!!!ИСПОЛЬЗУЕТСЯ КОГДА НЕ ИСПОЛЬЗУЕШЬ СБОРЩИКИ):

npm i -D live-server

в package.json в scripts написать "start": "live-server src"

!!!МЫ НЕ ИСПОЛЬЗУЕМ С НИМ СБОРЩИКИ ПОЭТОМУ В ФАЙЛЕ index.html ПОДКЛЮЧАЕМ ФАЙЛЫ css И script

!!!ЕСЛИ РАБОТАЕМ С МОДУЛЯМИ, Т.Е. СОБИРАЕМ ВСЕ В ОДНОМ ФАЙЛЕ js(index.js), ТО В index.html в теге script пишем еще type="module"
